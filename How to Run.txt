# HOW TO RUN THE BINARY IMAGE QNN PROJECT

This project has two parts: A Python (Django) Backend and a React Native (Expo) Mobile App.

---
### PART 1: RUN THE BACKEND (DJANGO)
1. Use Python 3.10.0 (64-bit).
2. Open a terminal in the main project folder (`BinaryImageQNN`).
3. Create a virtual environment:
   virtualenv -p python3.10 venv
4. Activate the virtual environment:
   - On Windows: venv\Scripts\activate
   - On Mac/Linux: source venv/bin/activate
5. Install backend dependencies:
   pip install -r requirements.txt
6. Start the Backend Server:
   python manage.py runserver 0.0.0.0:8000

---
### PART 2: RUN THE MOBILE APP (EXPO GO)
1. Open a *new* terminal window.
2. Go to the MobileApp folder:
   cd MobileApp
3. Install frontend dependencies:
   npm install
4. Start the Expo Server:
   npx expo start -c
5. Open the "Expo Go" app on your physical phone (Android/iOS).
6. Scan the QR code that appears in the terminal!

---
### PART 3: CONNECTING THE TWO
If you want the mobile app to talk to your local backend (instead of Render), go to `MobileApp/services/api.js` and change the `BASE_URL` to your computer's local Wi-Fi IP address (e.g., `http://192.168.1.5:8000`). Both your phone and laptop MUST be on the same Wi-Fi network.
